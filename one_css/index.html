About one.css

one.css is designed to provide a premium, modern look with minimal effort. It leverages semantic HTML to create high-quality interfaces without the need for complex class name systems.

Why one.css?

  • Classless by default: Style standard HTML elements directly.
  • Dark Mode & Themes: Built-in light dark support. Change the whole color scheme by adding a single class (e.g., .theme-blue) to the <html> element. (See the bottom of the sidebar for available themes; append the lowercase name to .theme-).
  • Premium Aesthetics: Modern typography (Inter), smooth transitions, and a curated color palette.
  • Responsive: A robust grid system that handles sidebar overlays and centered layouts.
  • 256 Icons Included: Hand-picked Material Symbols, covering 99% of web tool use cases.

Installation

Set up one.css in your project:

  1. Download the bundle (ZIP).
  2. Add the stylesheet to your <head>: 
    <link rel="stylesheet" type="text/css" href="one.css">
  3. Place the Inter and Material Symbols fonts in the same folder.
  4. Optional: Add one.js for toasts and dialogs: 
    <script src="one.js"></script>

Typography

Body copy is rendered in Inter (variable font, loaded locally from Inter-VariableFont_opsz,wght.ttf), with a full system-font fallback chain. Font smoothing (-webkit-font-smoothing: antialiased / -moz-osx-font-smoothing: grayscale) is applied globally so text looks crisp on every platform, including macOS retina screens.

Scroll Shadows

Use scroll shadow utility classes to indicate scrollable content boundaries. These utilities dynamically reveal top/bottom or left/right shadows as content overflows and scrolls.

Horizontal Scroll Shadows

Apply the .scroll-shadow-x class to a container. By default, it expects a background matching var(--surface-container). For containers with other background colors, define the --scroll-shadow-bg custom property.

Tooltips

Add a tooltip to any element by simply adding the data-tooltip="Your text here" attribute. Thanks to modern CSS anchor positioning, tooltips automatically adapt their position to always remain visible on-screen and close to their source.

Hover this text

To display a tooltip exclusively on mobile viewports (e.g., when text labels are hidden via .hide-mobile and only icons remain), use data-tooltip-mobile="Your text". This tooltip automatically disables itself on desktop screens (900px and up).
