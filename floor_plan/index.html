Floor Plan DSL Specification

        


        
A declarative DSL for constraint-based room layout optimization.


        

            Parameters:
            append weight=1.5
            to any rule to increase its importance (default 1
            ). Append required to mark a rule as a hard constraint — the optimizer
            retries until it is satisfied.
        


        
1. Global Settings

        

            

                

                    Syntax
                    Description
                

            

                
                    canvas 500 500
                    

                    canvas 500x500
                    

                    canvas 500 500 flexible
                		
                
                    Constrains layout to fixed bounds. Without flexible
                    , the floor plan is always compressed to fit the canvas. With
                    flexible, the canvas is a soft constraint — overflow is penalized
                    but not forced out.
                
            

            

                
                    seed 42
                
                Deterministic PRNG seed.
            

            

                
                    iter 2
                
                Multiplies the default iteration budget.
            

            

                
                    ratio_max 3:1
                
                Global maximum aspect ratio for any room.
            

            

                
                    area_min 100
                    

                    side_min 10
                    

                    side_min 10 flexible
                    

                    side_max 50
                		
                
                    Global area and side length constraints. side_min
                    is enforced as a hard constraint by default — room shape curves are clamped so
                    no dimension falls below the limit. Append flexible to treat it as
                    a soft hint only.
                
            

            

                
                    cwl 100
                    

                    cwc 2
                		
                
                    Default connected wall length (cwl
                    ) and min connected wall count (cwc).
                
            

            

        


        


        
2. Rooms & Variables

        

            

                

                    Syntax
                    Description
                

            

                
                    room A
                
                
                    Declare a room named A.
                
            

            

                
                    room A area=200
                    

                    room A area=10x20
                		
                Set target area or target dimensions for the room.
            

            

                
                    room A area_min=50 side_min=5
                
                Override global constraints for this room.
            

            

                
                    room A ratio=3:2
                
                Target aspect ratio for the room.
            

            

                
                    room A ratio_max=2:1
                
                Override global maximum aspect ratio for this room.
            

            

                
                    group = [A, B]
                
                Define a variable containing multiple rooms.
            

            

        


        


        
3. Rules & Constraints

        

            

                

                    Syntax
                    Description
                

            

                
                    A close B
                
                Minimize distance between rooms.
            

            

                
                    A far B
                
                Maximize distance between rooms.
            

            

                
                    A connect B
                
                
                    Rooms must share a wall. Append cwl=50 to override the required
                    wall length.
                
            

            

                
                    A {rule} any [B, C]
                    

                    A {rule} any list
                		
                
                    Target-side any
                    : OR
                    across targets — rule satisfied if it holds for at least one target. Works with
                    connect
                    , close
                    , and far.
                
            

            

                
                    any [A, B] {rule} C
                    

                    any list {rule} C
                		
                
                    Subject-side any
                    : OR
                    across subjects — rule satisfied if it holds for at least one room in the
                    subject list. Combine freely with target-side any
                    : any [A, B] connect any [C, D].
                
            

            

                
                    A {rule} [all but B]
                    

                    A {rule} [all but B, C]
                    

                    A {rule} [all but group]
                		
                
                    Expands to every declared room except those listed. Works as subject too: [all
                    but A] {rule} B. Exclusion is comma-separated rooms and/or group names.
                
            

            

                
                    A at north
                
                Snap room to the layout edge (north, south, east, west, edge).
            

            

                
                    A not at north
                
                Penalize touching the specified layout bound.
            

            

                
                    A enclosed
                
                Avoid placing the room on any outer edge.
            

            

        


        


        

            4. Nested Floor Plans (inside)
        

        

            

                

                    Syntax
                    Description
                

            

                

inside A {
  room x
  room y
  x connect y
}

                		
                
                    Defines a sub-floor-plan inside
                    room A.
                    The outer plan is optimized first. Once A's dimensions
                    are known, its inner plan is optimized using those dimensions as the canvas.
                    The inner scope is isolated — inner room IDs do not conflict with outer ones.
                
            

            

                
                    canvas inside a block
                
                
                    Error. The canvas for an inner plan is always the
                    resolved dimensions of the parent room. Declaring it explicitly is not allowed.
                
            

            

                
                    Nested inside
                
                
                    Works recursively. An inside block can itself contain
                    further inside blocks. Each level is optimized after
                    its parent's dimensions are resolved.
                
            

            

                

inside suite {
  room bed area=80000
  bed connect kitchen
  bed far loud
  bed close any meal
}

                		
                
                    Inner rooms may reference outer rooms and groups
                    as targets in close
                    , far
                    , and connect rules.
                    The outer room is treated as a fixed reference — its position relative to the
                    parent room is known after the outer plan is solved.

                    

                    A cross-boundary connect
                    is translated to at <dir>
                    : the direction (north/south/east/west) is inferred from the outer room's
                    position relative to the parent room, so the inner room is pushed to the nearest
                    wall facing that outer room.

                    

                    required is not supported on all cross-boundary rules and will be
                    ignored with a warning.
                    Outer rooms cannot be rule subjects
                    inside an inside block.
