Wireframe mockup markup language compiler & previewer

MockML Reference

A text-based markup language for building interactive wireframes and mockups.

1. Structure & Layout

Syntax Description
title: <text> Document title metadata (must be on first line). Example: title: My App
min_height: <num> Global minimum height metadata. Example: min_height: 500
'text' or "text" Plain cell label. Supports backslash escaping. Example: 'User Profile'
| Column separator. Lays out adjacent cells horizontally. Example: 'Left' | 'Right'
[ ... ] Table layout grid. Newlines inside the brackets create rows, and column separators | create columns.
{ ... } Container group. Wraps its inner content in a container.
spacer() Empty spacer cell that automatically expands to fill available layout space.
hr() Horizontal divider line.

2. Modifiers & Sizing

Modifier Description
:N Flex weight sizing hint. Example: 'Main':3 | 'Sidebar'
:h<size> Fixed height constraint. Example: 'Header':h80, 'Content':h20rem
:min-h<size> Minimum height constraint. Example: 'Footer':min-h100
:borders Visible borders. Keywords: all, l (left), r (right), u (top), d (bottom), lr (left/right), ud (top/bottom). Example: {a|b}:ud
:@align Text/flex alignment. Keywords: left/l, right/r, center/c, justify/j. Example: 'Center label':@center
:@valign Vertical/cross-axis alignment. Keywords: top/t, bottom/b, middle/m. Example: 'Top aligned':@top, 'Centered':@center@middle
[:borders ...] Table default borders. Applies modifiers to every child in the table grid. Example: [:ud 'Row 1' | 'Row 2']
Combined Combined modifier inline (alignment, weight, borders). Example: 'Title':@center2ud

3. UI Components

Syntax Description
button(text) Button element. Example: button(Cancel)
hbutton(text) Highlighted (primary) button. Example: hbutton(Submit)
button(icon(name)) Icon-only button (renders as btn-sym). Example: button(icon(download))
input(placeholder) Text input field. Example: input(Search...)
link(text)
link(url, text)		 Hyperlink. 1 arg: label text (href="#"). 2 args: URL (first) and label (second). Example: link(/home, Home)
img(alt)
img(url, alt)
img(..., ratio)		 Image placeholder. Supports optional aspect ratio hint (e.g. 1:1, 16:9). Example: img(avatar, 1:1)
select(o1, o2, ...) Dropdown selector with options. Example: select(Red, Green, Blue)
htag(text) Small colored status badge/pill. Example: htag(New)
tag(text) Chip/tag indicator. Example: tag(Active)
icon(name) Material icon. Example: icon(settings)

4. Annotations, Comments & Macros

Syntax Description
// comment Line comment. Must be outside quotes or brackets, and preceded by space or a column separator. Example: // sidebar cells
<- note text Descriptive side note box connected to the element with a line. Example: 'Stats' <- Refreshes daily
<<- note text Inline side note box targeting the specific preceding inline element or text. Example: button(Add) <<- 'Adds item'
sup(N)
sup(N) note text		 Footnote reference superscript, and footnote definition at the bottom of the code. Example: sup(1) Must be unique
$macro = { ... } Macro template definition. Calls wrap contents in a card. Example: $card = { $0 | $1 }
inline $macro = { ... } Inline macro template definition. Inserts contents transparently. Example: inline $box = { $0 }
$macro(args) Invocation of a macro template with parameter arguments. Example: $card(Username, input())
